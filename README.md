# 📜 NeowsForge 项目架构说明书 (V1.0)

## 1. 项目定位 (Project Mission)
**NeowsForge** 是一个专为《杀戮尖塔2》(Slay the Spire 2) 打造的**卡组性能沙盘推演器**。
它不只是简单的卡牌库查看器，而是一个基于数学期望（EV）模型的动态评估系统，旨在解决肉鸽游戏中“如何科学选牌”和“卡组发育节奏评价”的核心痛点。

## 2. 核心技术栈 (Technical Stack)
* **前端结构：** 纯 HTML5 / CSS3 / Vanilla JavaScript (无第三方依赖，轻量且高度可移植)。
* **数据驱动：** 依赖 `STS2_Card_Database_ZHS.json` 外部数据库进行渲染。
* **核心算法：** 线性概率期望模型 + 边际效用（$\Delta$ EV）演算引擎。

## 3. 三大交互范式 (Interaction Paradigm)
为了保持操作的连贯性，项目统一了全局交互逻辑：
1.  **左键单击 (Left Click)：** 触发“确定/流转”行为。在牌库是“加入”，在卡组/推演台是“移除”。
2.  **右键单击 (Right Click)：** 触发“状态切换/解析”行为。在卡组是“强化(Upgrade)”，在牌库是“发送至推演台分析”。
3.  **动态联动 (Dynamic Linking)：** 卡组（中间）的任何变动会即时触发仪表盘（右侧）和推演台（左侧）的重算。

## 4. 核心逻辑模块 (Core Logic)

### A. 期望值演算引擎 (EV Engine)
* **机制：** 基于 `(总数值 / 卡组总数) * 回合抽牌数`。
* **乘区模型：** * **力量/敏捷：** 通过正则提取文本（`match(/(获得|增加)\s*(\d+)\s*点力量/)`），按抽到对应牌型的概率进行线性增益计算。
    * **易伤/虚弱：** 易伤直接乘以 `1.5` 乘区；虚弱则动态削减 Boss 的目标攻击力（及格线下降）。

### B. 进度及格线系统 (Pacing Line)
* **全局层数逻辑：** 支持 0-48 层连续刻度。
* **锚点机制：** 根据选定 Boss 自动设置锚点（Act 1: 17层, Act 2: 33层, Act 3: 48层）。
* **及格线算法：** `及格线 = Boss 极值 * (当前层数 / 锚点层数)`。通过 CSS `left` 百分比驱动游标位移。

### C. 抓牌推演台 (Draft Analyzer)
* **边际效用检测：** 核心解决“稀释效应”。
* **计算公式：** `Δ EV = 加入新卡后的期望 - 当前卡组期望`。
* **智能诊断：** 自动识别“自过牌卡”对稀释的抵消作用，并给出绿色（神卡）、红色（稀释）等直观评价。

## 5. 文件结构 (File Structure)
* `index.html`：承载所有 UI 样式、HTML 结构及核心运算逻辑。
* `STS2_Card_Database_ZHS.json`：卡牌图纸库，包含名称、费用、数值及描述文本。
* `.gitignore`：严格屏蔽 `sts2/` 和 `SlayTheSpire2pck/` 等解包源文件夹。

## 6. 开发环境 (Environment)
* **本地开发：** 需要开启本地 Web 服务器（如 VS Code Live Server）以绕过浏览器的文件读取安全限制（CORS）。
* **外部数据：** 支持通过 Regex 实时解析卡牌描述文本，对 JSON 数据的依赖具有极强的鲁棒性。

### 💡 开发者接力提示：
* **添加新 Boss：** 只需在 `index.html` 的 `target-enemy` 下拉框中添加 `option`，并设置对应的 `data-dmg`（输出极值）和 `data-blk`（格挡极值）。
* **优化正则：** 如果发现某些 Buff 没识别出来，优先检查 `updateWorkshop()` 里的正则匹配式。
* **UI 扩展：** 新增功能面板应保持“原木/纸质”的自然护眼风格。
