📜 NeowsForge 项目架构说明书 (V1.1 解耦重构版)
1. 项目定位 (Project Mission)

NeowsForge 是一个专为《杀戮尖塔2》(Slay the Spire 2) 打造的卡组性能沙盘推演器。

它不只是简单的卡牌库查看器，而是一个基于数学期望（EV）模型的动态评估系统，旨在解决肉鸽游戏中“如何科学选牌”和“卡组发育节奏评价”的核心痛点。

2. 核心技术栈 (Technical Stack)

前端架构： 纯 HTML5 / CSS3 / Vanilla JavaScript (无第三方依赖，轻量且采用标准的三端分离结构)。

数据驱动： 依赖 STS2_Card_Database_ZHS.json 外部数据库进行实时渲染。

核心算法： 线性概率期望模型 + 边际效用（$\Delta$ EV）演算引擎。

3. 三大交互范式 (Interaction Paradigm)

为了保持操作的连贯性，项目统一了全局交互逻辑：

左键单击 (Left Click)： 触发“确定/流转”行为。在牌库是“加入卡组”，在卡组/推演台则是“移除卡牌”。

右键单击 (Right Click)： 触发“状态切换/解析”行为。在卡组和推演台是“绿字强化(Upgrade)”，在牌库是“一键发送至推演台分析”。

动态联动 (Dynamic Linking)： 主卡组（中间区）的任何变动，会即时触发仪表盘（右侧）期望的重算，以及推演台（左侧）稀释率差值的动态跟随。

4. 核心逻辑模块 (Core Logic)
   A. 期望值演算引擎 (EV Engine)

基础机制： 基于 (总数值 / 卡组总数) * 回合抽牌数。

乘区模型： * 力量/敏捷： 通过正则提取文本（如 match(/(获得|增加)\s*(\d+)\s*点力量/)），按实际抽到对应牌型（攻/防）的概率进行线性乘法增益。

易伤/虚弱： 易伤提供独立的 1.5 倍增伤乘区；虚弱则动态削减 Boss 的目标攻击力（使防守及格线下降）。
B. 进度及格线系统 (Pacing Line)

全局层数逻辑： 支持 0-48 层连续滑块刻度。

锚点机制： 根据选定 Boss 自动设定极限锚点（Act 1: 17层, Act 2: 33层, Act 3: 48层）。

及格线算法： 当前节点及格线 = Boss 极值 * (当前层数 / 锚点层数)。通过 CSS 变量驱动游标在进度条上平滑位移。

C. 抓牌推演台 (Draft Analyzer)

边际效用检测： 核心解决卡牌游戏中的“稀释效应（Dilution Effect）”。

计算公式： Δ EV = 加入新卡后的新卡组期望 - 原卡组期望。

智能诊断： 自动识别“自过牌属性”对稀释率的完美抵消，并给出绿色（全面拉升）、红色（严重稀释）等直观评价，证明了“跳过（Skip）”的数学价值。

5. 文件结构 (File Structure)

项目已完成解耦，采用高内聚、低耦合的模块化结构：

index.html (骨架)： 承载所有的 UI 布局和 DOM 节点定义，作为核心视图层。

style.css (皮囊)： 统一管理“原木/护眼”自然风格的视觉表现，以及进度条、动画特效的控制。

app.js (灵魂)： 集中处理 JSON 数据抓取、卡牌组件生成、DOM 事件绑定以及所有的核心 EV 数学演算逻辑。

STS2_Card_Database_ZHS.json： 卡牌图纸库源文件，包含全部解析数据的中文化结构。

.gitignore： 门卫文件，严格屏蔽 sts2/ 和 SlayTheSpire2pck/ 等几十G的解包源文件夹，保障 Git 仓库的纯净与合法性。

6. 开发环境 (Environment)

本地开发： 需要开启本地 Web 服务器（如 VS Code 的 Live Server 插件）以绕过浏览器的文件读取安全限制（CORS）。

数据鲁棒性： 支持通过 Regex 实时解析卡牌描述文本，极大地降低了对 JSON 静态结构绝对精准度的依赖。

💡 开发者接力提示：

寻找逻辑代码： 所有的事件处理（如 updateWorkshop()）和卡牌渲染（如 createCardButton()）现在都在 app.js 中，不要去 HTML 里找了。

修改视觉样式： 如需调整游标大小、卡牌颜色，请直接前往 style.css 修改对应类名。

添加新 Boss 环境： 请在 index.html 的 #target-enemy 下拉框中添加新的 <option>，并正确赋予 data-act, data-dmg, data-blk 参数即可，JS 引擎会自动读取计算。

优化正则表达式： 如果发现后续新 DLC 或补丁中的特殊 Buff 未被识别，优先检查 app.js 遍历循环里的正则匹配式。
